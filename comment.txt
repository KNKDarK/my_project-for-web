========== ROOT CONFIGURATION FILES ==========

Caddyfile:
Server configuration for reverse proxy and routing, handles HTTPS and request forwarding to the Next.js application.

CNAME:
DNS configuration file specifying the domain name for the deployed web app (e.g., custom domain routing).

package.json:
Defines project metadata, npm dependencies (Next.js, React, Prisma, NextAuth, shadcn UI, Tailwind), and scripts for dev/build/start.

pnpm-lock.yaml:
Lock file for pnpm package manager ensuring consistent dependency versions across environments and installations.

pnpm-workspace.yaml:
Monorepo workspace configuration allowing multiple packages to be managed together in one workspace structure.

next.config.ts:
Next.js configuration enabling standalone output mode, ignoring build errors, and disabling strict mode for production builds.

tsconfig.json:
TypeScript configuration specifying compiler options, path aliases, and JavaScript target version for the entire project.

postcss.config.mjs:
PostCSS configuration enabling Tailwind CSS and Autoprefixer for CSS processing and vendor-prefix handling.

tailwind.config.ts:
Tailwind CSS configuration defining custom theme, colors, spacing, and component styling rules for the app.

eslint.config.mjs:
ESLint configuration for code quality checks, enforcing coding standards and best practices across the codebase.

middleware.ts:
Next.js middleware functions handling authentication redirects, request validation, and protected route enforcement.

next-env.d.ts:
TypeScript declaration file auto-generated by Next.js for type definitions and Next.js specific types.

README.md:
Project documentation with setup instructions, build commands, and overview of the web application purpose.

components.json:
shadcn UI configuration file defining component paths and import aliases for UI component scaffolding.

========== HOW FILES AFFECT THE WEB APP ==========

IMPACT SUMMARY:
- Config files (next.config, tsconfig, tailwind.config) establish build, type, and styling foundation.
- Prisma schema + migrations create database for storing users, posts, and bookings data.
- NextAuth (auth.ts + middleware) protects admin routes and enables credentials-based login.
- Layout.tsx and globals.css provide global page structure, fonts, and baseline styling.
- Page.tsx implements interactive homepage with portfolio, animations, and contact form.
- Admin bookings page manages booking entries with CRUD operations and status filtering.
- UI components (button, card, input, etc.) provide consistent, accessible design system throughout app.
- Hooks (use-mobile, use-toast) enable responsive behavior and notifications.
- Database client (db.ts) connects to PostgreSQL for CRUD operations across all pages.
